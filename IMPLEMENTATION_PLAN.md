## Implementation Pattern

Each CLI module follows this pattern:

```python
# cli/<domain>.py
import typer
from uk_parliament_mcp.config import <DOMAIN>_API_BASE
from uk_parliament_mcp.http_client import get_result, build_url
from uk_parliament_mcp.cli.utils import run_async, format_output

app = typer.Typer(help="<Domain> commands")

@app.command("command-name")
def command_function(
    param: str = typer.Argument(..., help="Parameter description"),
    pretty: bool = typer.Option(False, "--pretty", "-p"),
    data_only: bool = typer.Option(False, "--data-only", "-d"),
):
    """Command description."""
    url = f"{<DOMAIN>_API_BASE}/endpoint?param={param}"
    result = run_async(get_result(url))
    typer.echo(format_output(result, pretty, data_only))
```

**Key Principles:**
1. Reuse MCP http_client logic (no duplication)
2. Consistent output formatting via utils.format_output()
3. Standard flags: --pretty, --data-only
4. Direct mapping from MCP tool function to CLI command
5. Preserve query parameters and URL patterns from MCP tools
