## Notes

- Each task is one API module — all tools for that module in one iteration
- Follow existing patterns in each `tools/*.py` and `cli/*.py` file
- Reference `AddMissingTools.md` for endpoint URLs and details
- Reference OpenAPI specs in `context/` directory for parameter details
- **SI v1 base URL issue:** `STATUTORY_INSTRUMENTS_API_BASE` is v2 only. New v1 endpoints need `STATUTORY_INSTRUMENTS_API_BASE_V1 = "https://statutoryinstruments-api.parliament.uk/api/v1"` defined locally or in a shared config.
- **WhatsOn URL construction:** `WHATSON_API_BASE` ends at `/calendar`, so new endpoints like `/calendar/categories/list.json` become `{WHATSON_API_BASE}/categories/list.json` (drop the leading `/calendar`).
- **Sessions URL quirk:** Session endpoints use `{WHATSON_API_BASE}/sessions/byid.json/{sessionId}` — the `.json` suffix comes before the path parameter.
- **Hansard speakers-for-day:** Requires `section` as a required query parameter in addition to the path params.
- **AddMissingTools.md vs missing-tools-spec.md discrepancy:** The spec lists 8 members tools; `AddMissingTools.md` lists 10 (adds `browse_locations` and `get_lords_interests_register`). This plan includes all 10.
- **Committees discrepancy:** Spec lists 5 committee tools but `get_broadcast_meetings` is already done; net is 4 new tools.
- **Total new tools:** 10 members + 10 hansard + 11 whatson + 8 SI + 3 interests + 4 committees + 1 now = 47 new tools, targeting ~210 total.
